Почему HTML считает, что «CANDY» и «BruceWillis» — это цвет?

Вы никогда не пробовали задавать цвет в HTML при помощи разных строк? Например, строка
‹font color="#chucknorris"›chucknorris‹/font›
Пропускаемые цифры распознаются как 0[...]. Некорректные символы распознаются как 0. Так что, например, значения #F0F0F0, F0F0F0, F0F0F, #FxFxFx и FxFxFx – одно и то же.
    Если применить эти правила, то получим следующее:

  • Заменяем все неправильные шестнадцатеричные символы нулями 0, chucknorris превращается в c00c0000000
  • “Добиваем” число нулями до количества знаков, делимого на 3 (11 -> 12), получаем c00c00000000
  • Разбиваем на три группы, каждая из которых отвечает за одну компоненту RGB: RGB (c00c, 0000, 0000)
  • Обрезаем каждый из аргументов справа, оставляя 2 символа, и получаем наш результат RGB (c0, 00, 00) = #C00000 или он же RGB(192, 0, 0).
‹font color="#BruceWillis"›BruceWillis‹/font›
BruceWillis

=

‹font color="#B0e000"›BruceWillis‹/font›
BruceWillis
‹font color="#CANDY"›CANDY‹/font›
CANDY

=

‹font color="#ca0000"›CANDY‹/font›
CANDY
  • Заменяем все неправильные шестнадцатеричные символы нулями 0, BruceWillis превращается в b00ce000000
  • “Добиваем” число нулями до количества знаков, делимого на 3, получаем
    b000ce000000
  • Разбиваем на три группы, каждая из которых отвечает за одну компоненту RGB:
    RGB (b00c, e000, 0000)
  • Обрезаем каждый из аргументов справа, оставляя 2 символа, и получаем наш результат
    RGB (b0, e0, 00) = #B0e000
  • Заменяем все неправильные шестнадцатеричные символы нулями 0, CANDY превращается в
    ca0d0
  • “Добиваем” число нулями до количества знаков, делимого на 3, получаем
    ca0d00000000
  • Разбиваем на три группы, каждая из которых отвечает за одну компоненту RGB:
    RGB (ca0d, 0000, 0000)
  • Обрезаем каждый из аргументов справа, оставляя 2 символа, и получаем наш результат
    RGB (ca, 00, 00) = #ca0000
